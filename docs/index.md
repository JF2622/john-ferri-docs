# Welcome to MkDocs

For full documentation visit [mkdocs.org](https://www.mkdocs.org).

## Portfolio Overview
This repository demonstrates my ability to create and maintain operational documentation for a Security Operations Center (SOC) using a modern Docs-as-Code workflow.

## What This Repository Demonstrates
- SOC technical writing (SOP + playbooks)
- Clear, actionable triage steps and escalation criteria
- MkDocs site structure and publishing via GitHub Pages
- Git workflow: feature branches, pull requests, and iterative updates
- Cross-platform contributor experience (Windows and WSL support scripts)

## Start Here
- **SOP:** Tier 1 SOC Onboarding (baseline workflow and standards)
- **Playbooks:** Triage guides for common alert scenarios
- **Style Guide:** Writing standards used across the docs

## Local Preview
For local preview instructions (Windows and WSL), please see the repository README on GitHub.


## Commands

* `mkdocs new [dir-name]` - Create a new project.
* `mkdocs serve` - Start the live-reloading docs server.
* `mkdocs build` - Build the documentation site.
* `mkdocs -h` - Print help message and exit.
